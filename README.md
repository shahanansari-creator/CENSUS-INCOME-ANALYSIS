## 🧭 Project Overview

This project answers two core questions using the Census Income dataset:

1. **What does the data tell us?** — Through cleaning, slicing, filtering, grouping, and visualization, we uncover patterns in income across education, occupation, work hours, marital status, and more.
2. **Can we predict income class?** — Using Logistic Regression, Decision Tree, and Random Forest classifiers, we predict whether an individual earns `<=50K` or `>50K` per year, and compare model performance.

A Linear Regression model is also included to study the relationship between education level and weekly working hours.

**Goal:** Demonstrate a complete, reproducible data science pipeline — from raw, messy data to a compared set of trained models — in a single, well-documented, "download-and-run" notebook.

## 🗂 Dataset

**File:** `census-income.csv`
**Source:** Adapted from the UCI Machine Learning Repository's "Adult" dataset, based on 1994 US Census Bureau data.
**Size:** 32,561 rows × 15 columns (30,162 rows after cleaning)

| # | Column | Type | Description |
|---|--------|------|-------------|
| 1 | `age` | Numeric | Age of the individual |
| 2 | `workclass` | Categorical | Type of employer (Private, Self-emp, Government, etc.) |
| 3 | `fnlwgt` | Numeric | Census sampling weight |
| 4 | `education` | Categorical | Highest level of education attained |
| 5 | `education-num` | Numeric | Education level, encoded numerically |
| 6 | `marital-status` | Categorical | Marital status |
| 7 | `occupation` | Categorical | Occupation type |
| 8 | `relationship` | Categorical | Household relationship role |
| 9 | `race` | Categorical | Race |
| 10 | `sex` | Categorical | Gender |
| 11 | `capital-gain` | Numeric | Income from investments |
| 12 | `capital-loss` | Numeric | Investment losses |
| 13 | `hours-per-week` | Numeric | Hours worked per week |
| 14 | `native-country` | Categorical | Country of origin |
| 15 | `annual_income` | Categorical (Target) | `<=50K` or `>50K` |

> ⚠️ **Note on missing data:** Missing values are encoded as `"?"` rather than blank cells, concentrated in `workclass`, `occupation`, and `native-country`.

## 🚀 Getting Started

### Option 1: Run in Google Colab (recommended)

1. Click **File ▸ Upload notebook** in [Google Colab](https://colab.research.google.com/) and select `Census_Income_Analysis_Project.ipynb`, **or** open it directly from GitHub via `File ▸ Open notebook ▸ GitHub` and paste this repo's URL.
2. Run all cells: `Runtime ▸ Run all`.
3. When prompted, upload `census-income.csv` (only required if it isn't already in the Colab working directory).

That's it — the notebook is fully self-contained and installs any missing packages automatically.

### Option 2: Run locally

```bash
# Clone the repository
git clone https://github.com/<your-username>/census-income-analysis.git
cd census-income-analysis

# (Recommended) create a virtual environment
python3 -m venv venv
source venv/bin/activate        # On Windows: venv\Scripts\activate

# Install dependencies
pip install pandas numpy matplotlib seaborn scikit-learn jupyter

# Launch the notebook
jupyter notebook Census_Income_Analysis_Project.ipynb
```

## 🤖 Machine Learning Models & Results

| Model | Target | Predictors | Split | Accuracy |
|-------|--------|-----------|:---:|:---:|
| Linear Regression | `hours-per-week` | `education-num` | 70:30 | R² = 0.028 |
| Logistic Regression (Simple) | `income class` | `occupation` | 65:35 | **75.11%** |
| Logistic Regression (Multiple) | `income class` | `age`, `workclass`, `education` | 80:20 | **78.52%** |
| Decision Tree | `income class` | All features | 70:30 | **81.09%** |
| **Random Forest (300 trees)** | `income class` | All features | 80:20 | **🏆 85.41%** |

## 💡 Key Insights

- ✅ ~7.4% of records had missing data, concentrated in `workclass`, `occupation`, and `native-country`.
- ✅ **80.17%** of individuals in the dataset are married.
- ✅ **51.70%** of the dataset consists of US-based private-sector employees earning `<=50K`.
- ✅ Only **16.43%** of high school graduates earn `>50K`, versus much higher rates for those with a Bachelor's degree or higher.
- ✅ A single-feature model (occupation only) achieves deceptively "good" accuracy (75.11%) while **never predicting the minority class** — a cautionary example of why accuracy alone is insufficient on imbalanced datasets.
- ✅ Ensemble methods (Random Forest) clearly outperform single models (Decision Tree, Logistic Regression) on this structured, mixed-type tabular data.

## ⚠️ Limitations & Future Work

- **Class imbalance** (~3:1 toward `<=50K`) affects evaluation; accuracy alone can overstate real-world usefulness.
- **Listwise deletion** of missing values discards data rather than imputing it.
- **Label encoding** on tree models imposes an artificial ordinal relationship on unordered categories.
- The dataset reflects **1994 labor market conditions** and may not generalize to today's economy.

**Planned improvements:**
- [ ] Handle class imbalance via class weighting, SMOTE, or threshold tuning
- [ ] Add precision, recall, F1-score, and ROC-AUC evaluation
- [ ] Hyperparameter tuning via `GridSearchCV` / `RandomizedSearchCV`
- [ ] Try gradient boosting methods (XGBoost, LightGBM)
- [ ] Feature engineering (e.g., net capital change, binned age/hours)
